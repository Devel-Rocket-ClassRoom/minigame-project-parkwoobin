## 1. 모바일 입력 UI (가상 게임패드)

### 목표
Android에서 키보드 없이 모든 조작이 가능한 터치 버튼 구성.

### 버튼 구성

| 버튼 | 기능 | 위치 |
|------|------|------|
| 좌 / 우 | 이동 | 왼쪽 하단 |
| 점프 (A) | 점프 | 오른쪽 하단 |
| 대시 (B) | 대시 | 오른쪽 하단 |
| 숙이기 (C) | 숙이기 토글 | 오른쪽 하단 |
| 공격 (D) | 공격 | 오른쪽 하단 |
| 상호작용 (F) | 박스 열기 등 | 오른쪽 하단 |

### 작업 항목

- [ ] UI Canvas (Screen Space — Overlay) 생성, `CanvasScaler` 1080×1920 세로 설정
- [ ] 좌/우 이동 버튼: `EventTrigger` PointerDown/PointerUp → `PlayerController.OnMove()` 호출
- [ ] 점프 버튼: PointerDown → PressJump / PointerUp → ReleaseJump (가변 점프 유지)
- [ ] 대시·숙이기·공격 버튼: PointerDown 이벤트 → 각 OnXxx 메서드 호출
- [ ] 상호작용 버튼: `BoxController`의 F키 폴링 → 버튼 이벤트로 교체
- [ ] `PlayerController.Input.cs`의 `ReadVerticalInput()` — 사다리용 상하 버튼 추가 또는 이동 버튼 재활용
- [ ] PC와 모바일 입력이 동시에 동작하도록 유지 (에디터 테스트 편의)

### 주의
- 점프 버튼은 `isPressed` 지속 감지가 필요하므로 단순 `onClick`이 아닌 `PointerDown / PointerUp` 이벤트로 구현
- 버튼 투명도 조절로 게임 화면 가림 최소화

## 2. 배고픔 시스템 고도화

### 현재 상태
- `HungerSystem`: 시간 감소·회복 동작 중
- 배고픔 0이 되어도 패널티 없음 (미완성)

### 목표
배고픔 0 → 체력 지속 감소 → 사망 연결까지 완성.

## 3. 경험치 시스템

### 목표
아이템 획득·적 처치·사람 회피 성공 시 경험치 누적 → 100% 달성 시 스테이지 클리어.

### 신규 스크립트: `ExpManager`

```
Assets/Scripts/Systems/ExpManager.cs
```

### 기능 명세

| 기능 | 내용 |
|------|------|
| 경험치 누적 | `AddExp(int amount)` 호출로 증가 |
| 100% 달성 | `GameManager.Instance.StageClear()` 호출 |
| 이벤트 | `OnExpChanged(int current, int max)` — HUD 연동용 |

### 경험치 획득 기준 (GDD 기준)

| 행동 | 경험치 |
|------|--------|
| 음식 아이템 획득 | +15 |
| 코인 획득 | +5 |
| 고양이 적 처치 | +30 |
| 사람 적 처치 | +20 |

## 4. 저장 시스템

### 목표
스테이지 클리어·게임오버 시 자동 저장. 게임 재시작 시 이전 진행 불러오기.

### 신규 스크립트: `SaveManager`

```
Assets/Scripts/Core/SaveManager.cs
```

### 저장 데이터 구조 (GDD 기준)

```csharp
[System.Serializable]
public class SaveData
{
    public int bestScore;
    public int totalMoney;
    public int unlockedStage;
    public int speedLevel;
    public int healthLevel;
    public int hungerLevel;
    public int feverLevel;
}
```

### PlayerPrefs 키 상수 (하드코딩 금지)

```csharp
public static class SaveKeys
{
    public const string BestScore     = "best_score";
    public const string TotalMoney    = "total_money";
    public const string UnlockedStage = "unlocked_stage";
    public const string SpeedLevel    = "speed_level";
    public const string HealthLevel   = "health_level";
    public const string HungerLevel   = "hunger_level";
    public const string FeverLevel    = "fever_level";
}
```

### 저장 타이밍

| 타이밍 | 트리거 |
|--------|--------|
| 스테이지 클리어 | `GameManager.OnStateChanged → StageClear` |
| 게임오버 | `GameManager.OnStateChanged → GameOver` |

- [ ] `Load()` — PlayerPrefs에서 읽어 각 시스템에 적용

## 5. 프롤로그 연출

### 목표
게임 시작 시 "사람 → 고양이" 변신 과정을 텍스트·연출로 표현.  
별도 씬 없이 현재 SampleScene 안에서 코루틴으로 처리 (씬 분리는 3차 이후).

### 연출 흐름

```
게임 시작 (GameState.Prologue)
  → 검은 화면 Fade In
  → 자막 순서 출력 (TypeWriter 효과)
     "골목을 걷는다..."
     "의문의 사건 발생"
     "정신을 잃음"
     "낡은 박스 안에서 깨어남"
     "앞발을 본다..."
  → 고양이 Idle 애니메이션 재생 (플레이어 오브젝트 활성화)
  → 배고픔 UI Fade In
  → Fade Out → GameState.Playing 전환
```
