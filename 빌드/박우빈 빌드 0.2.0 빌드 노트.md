## 2. v0.2 신규 구현 기능

### 2-1. 벽 그립 시스템 (Wall Grip)

플레이어가 공중에서 벽에 닿을 때 동작하는 매달리기 로직이 전면 재설계됨.

**벽 방향키를 누를 때 (pressingInto)**
- 중력 제거 + 수직속도 0 고정 → 벽에 매달린 상태 유지
- 벽 방향으로 스프라이트가 향함 (방향 잠금)

**벽 반대 방향키를 누를 때 (pressingAway)**
- 0.1초 유예 타이머(`_wallGripTimer`) 시작
- 유예 시간 동안 `gravityScale = 0`, `-wallNormal × 2f` 속도로 벽을 향해 밀기 → 물리 접촉 유지
- 타이머 만료 후 낙하 시작 (`_wallGripUsed = true`로 소모 표시)
- 벽 방향키를 다시 누르면 타이머/소모 초기화 → 재매달리기 가능
- 유예 시간 동안 스프라이트는 계속 벽 방향을 향함 (반대키 눌러도 뒤집히지 않음)

**one-shot 보장 (`_wallGripUsed` 패턴)**
- 반대방향키를 계속 누르고 있어도 타이머가 무한 재시작되지 않음
- 타이머가 한 번 만료되면 `_wallGripUsed = true` → 벽 방향키로만 초기화됨
- 착지 시 자동 초기화

---

### 2-2. 벽 점프 개선

| 항목 | 내용 |
|------|------|
| 판정 버퍼 | 벽 반대 방향키를 누른 상태에서 점프해도 그립 타이머 소모 → 벽 점프 성공률 향상 |
| 점프 후 0.25초 | `_wallJumpTimer` 동안 FixedUpdate가 수평속도 덮어쓰기 차단 |
| 벽 방향 입력 차단 | `_isWallJumping = true` 상태에서 벽 쪽 방향키 입력 무효화 (튕겨나가는 방향만 허용) |
| 착지 체크 | `_isGrounded && velocity.y <= 0f` 조합으로 상승 중 오판정 방지 |

---

### 2-3. 공중 대시 직선 운동

이전에는 대시 중 포물선으로 낙하하는 문제가 있었음.

- `_isDashing && !_isGrounded` 상태에서 `gravityScale = 0`, `velociy.y = 0` 고정 → **완전한 직선 대시**
- 대시 종료 시: 벽 접촉 중이면 벽 그립으로 전환, 공중이면 `velocity.y = -2f`로 자연스러운 낙하 시작

---

### 2-4. 대시 후 벽 그립 (`_dashWallStick`)

공중 대시 직후 벽에 닿아 있을 때 자동으로 벽에 고정됨.

- `_dashJustEnded` (한 프레임 플래그)로 대시 만료 직후 1회만 체크
- `_isOnWall && !_isGrounded` 조건 충족 시 `_dashWallStick = true`
- 고정 중: `gravityScale = 0`, `velocity = zero` → 벽에 붙어 정지
- 이 상태에서 벽 점프 / 착지 / 벽 이탈 시 자동 해제

---

### 2-5. 계단 1타일 벽 판정 제거 (Raycast 기반)

1칸짜리 턱(계단)에서 벽에 매달리는 현상을 방지하기 위한 Raycast 감지.

```
플레이어 위치에서 아래로 레이캐스트 → 지면 Y 계산
지면 Y + 1.1f 높이에서 벽 방향으로 레이캐스트(0.5f)
벽이 없으면 → 계단으로 판단 → _isOnWall = false
```

- Physics Material(friction=0)만으로는 계단 감지 불가 → 두 방식 병행
- 감지 기준점이 `feetY`가 아닌 `groundY + 1.1f`임에 주의 (플레이어 높이 무관)

---

### 2-6. 물리 재료 (PhysicsMaterial2D)

`Assets/Physics/PlayerNoFriction.physicsMaterial2D`

| 속성 | 값 |
|------|----|
| Friction | 0 |
| Bounciness | 0 |

- 플레이어 Collider2D에 적용
- 벽에 닿을 때 마찰력으로 인한 공중 부양 현상 해결 (점프 중 벽 방향키 누를 때)
- 이 Material 없이 코드만으로 공중 부양 차단 시도했다가 실패 이력 있음 → 제거 금지

---

### 2-7. Turn(회피) 모션 무적 및 총알 통과

**무적 처리**
- `TriggerTurn()` 호출 시 `_invincibleTimer = Mathf.Max(_invincibleTimer, turnDuration)` 설정
- Turn 모션 지속 시간(기본 0.5초) 동안 `TakeDamage`가 `false` 반환 → 피해 없음
- 피격 무적이 이미 남아있는 경우 더 긴 쪽을 유지 (`Mathf.Max` 사용 이유)

**물리 충돌 무시**
- `TurnPassthroughRoutine` 코루틴: Turn 시작 시 Player↔Enemy 레이어 충돌 비활성화
- Turn 종료 시 자동 복원

**총알 통과**
- `TakeDamage`가 `bool` 반환으로 변경됨
  - 실제 데미지가 들어간 경우 `true` → `Bullet.cs`가 총알 소멸
  - 무적 상태라 데미지 없는 경우 `false` → **총알이 플레이어를 통과해서 계속 날아감**
- Turn 회피 시 시각적으로 "피했다"는 연출이 가능

---

### 2-8. 아이들 애니메이션 무한 루프

이전에는 Idle 애니메이션이 3회 반복 후 마지막 프레임에서 정지(freeze)됨.

- `_idleFrozen`, `_lastUnfreezeFrame` 필드 제거
- idle freeze 블록 전체 제거 (`anim.speed = 0f` 로직 삭제)
- `Unfreeze()` 메서드 및 모든 호출 제거
- `AnyActionTriggerPending()` 및 `_actionTriggers` 배열 제거
- 결과: Idle 애니메이션이 멈추지 않고 계속 루프

### 벽 관련 조작 요령

| 상황 | 동작 |
|------|------|
| 벽에 매달리기 | 공중에서 벽에 닿은 채로 벽 방향키 유지 |
| 벽 점프 | 벽에 매달린 상태에서 Jump (방향키 상관없음) |
| 회피 점프 | 벽 반대 방향키를 0.1초 이내에 누른 후 Jump → 유예 시간 내 판정 |
| 공중 대시 후 벽 고정 | 대시가 벽에 닿으면서 끝나면 자동으로 벽에 고정 |

### 기본 플레이 흐름

1. 씬 시작 → `Playing` 상태 자동 전환
2. 적을 피하거나 공격하며 이동
3. 아이템 획득으로 배고픔 및 HP 유지
4. 열쇠 수집 → 상자 오픈 → 희귀 아이템 획득
5. HP 0 → GameOver

### 생존 팁

- 인간 적(Type1, Type3)은 **정면에서만 총을 쏨** → 뒤로 돌아서 접근
- **Turn 모션은 무적 + 총알 통과** → 총알이 날아올 타이밍에 Turn으로 회피 가능
- 고양이 Fast 타입은 **분노 상태에서 돌진** → 점프로 회피
- 초콜릿은 배고픔을 줄이므로 **획득 금지**
- 1칸짜리 턱에서는 벽 매달리기가 의도적으로 비활성화됨 (계단 감지 로직)

## 7. 알려진 제한 및 주의사항

### 벽 그립 관련

- **계단 감지 Raycast 기준: `groundY + 1.1f`**  
  플레이어 위치에서 지면까지 레이캐스트 후, 지면 Y에 1.1 unit을 더한 높이로 벽 감지.  
  타일 크기가 1 unit 기준이며, 타일 크기 변경 시 수치 재조정 필요.

- **벽 판정 법선 기준: `normal.x > 0.8`**  
  경사가 큰 타일에서 오판정 가능. 경사 타일 사용 시 확인 필요.

- **벽 그립 타이머 0.1초**  
  너무 짧으면 반응 안 됨, 너무 길면 허공에 떠있는 느낌. Inspector에서 `turnDuration`은 변경하되 `_wallGripTimer`는 코드 직접 수정 필요 (현재 하드코딩). -> 벽 점프 난이도 개선

### Turn 무적 관련

- **`turnDuration = 0.5f`** (Inspector `Action Cooldowns` 헤더 아래 `Turn Duration`)  
  이 값이 무적 시간과 Turn 모션 길이를 동시에 결정함.  
  애니메이션 클립 길이와 반드시 일치시킬 것.

- **Turn 중 총알은 통과하지만 소멸하지 않음**  
  총알이 플레이어를 지나쳐 날아가므로 뒤에 다른 오브젝트가 있으면 피격 가능.

- **Turn은 지상 또는 상승 중에만 발동** (`!_isGrounded && !IsAscending` 조건으로 차단)  
  낙하 중에는 Turn 입력이 무시됨.

### `TakeDamage` 반환값

- `void → bool`로 변경됨. **`TakeDamage`를 호출하는 다른 스크립트가 있다면 반환값 무시해도 동작은 정상이나, 총알 같이 "피해가 들어갔을 때만 소멸" 로직이 필요한 경우 반드시 반환값 체크 필요.**

### 물리 관련

- `PlayerNoFriction.physicsMaterial2D` 제거 시 점프 중 벽 방향키 누를 때 공중 부양 현상 재발
- 플레이어 공격 HitBox: 같은 프레임 동일 적 다중 피격 방지 처리됨

### 애니메이션

- Eat / Sleep 재생 중 이동·점프 입력 잠금은 의도적 동작

### 빌드 시 주의

- Android: `InputSystem_Actions.inputactions` 에셋 포함 확인
- URP 설정 파일(`Assets/Settings/`) 빌드에 포함 필수
- `PlayerNoFriction.physicsMaterial2D` (`Assets/Physics/`) 빌드에 포함 필수
- Bullet 프리팹 Material 미설정 시 매젠타(핑크) fallback → "Universal Render Pipeline/2D/Sprite-Unlit-Default" 셰이더 확인
