## 2. 이번 버전 구현된 기능

### 2-1. 플레이어 조작

| 기능 | 상세 |
|------|------|
| 좌우 이동 | `moveSpeed = 4f` 기본값, Input System `OnMove` 콜백 |
| 가변 점프 | 버튼을 오래 누를수록 높이 올라감. 최대 높이 2.5u / 최소 높이 0.4u |
| 낙하 가속 | 낙하 시 중력 3배 적용 (`fallMultiplier = 3.0f`) |
| 대시 | 지면 또는 점프 상승 중에만 가능. 지속 시간 0.2s, 힘 12f |

| 사다리 이동 | Ladder 레이어 OverlapPoint 감지. 중력 제거 후 수직 이동 |

### 2-3. 적 AI

#### 사람(HumanEnemy) — 3종 변형

| 변형 | HP | 속도 | 공격력 | 원거리 |
|------|----|------|--------|--------|
| Type1 | 4 | 2.2 | 1 | 총(쿨 2.5s) |
| Type2 | 6 | 2.0 | 2 | 없음 (근접 3종 패턴) |
| Type3 | 3 | 2.7 | 1 | 총(쿨 2.5s) |

- **상태**: Idle → Patrol → Chase → MeleeAttack / Aim → Hit
- **감지**: 범위(`detectionRange = 6f`) 내 진입 시 추격 시작
- **추격 포기**: 스폰 위치에서 `leashRange = 5f` 벗어나면 복귀
- **chase walk/run 전환**: 달리기 1.2–2.5s → 걷기 0.8–1.6s 주기로 반복
- **원거리(Type1, 3)**: 조준선(LineRenderer) → 선이 플레이어까지 뻗은 뒤 총알 발사
- **점프**: 벽 앞 또는 플레이어 높이 차이가 클 때 자동 점프 + 랜덤 점프(`1.2%`)

#### 고양이(CatEnemy) — 3종 변형

| 변형 | 색상 | HP | 속도 | 공격력 |
|------|------|----|------|--------|
| Normal | 주황 | 3 | 1.8 | 1 |
| Fast | 갈색 | 2 | 2.5 | 1 |
| Strong | 검정 | 6 | 1.2 | 2 |

- **상태**: Idle → Patrol → Chase → Hit
- **공격**: HitBox 범위 안에 플레이어가 들어오면 정지 후 Attack 트리거
- **피격 후 분노**: 60% 확률로 5–9s 동안 이동속도·감지범위 증가
- **착지 예측**: 0.8u 이내 접근 시 Land 모션 미리 재생

### 2-4. 아이템 시스템

#### FoodItem (Trigger 충돌 방식)

플레이어가 닿으면 즉시 효과 적용 후 파괴.

| Inspector 필드 | 용도 | 권장값 예시 |
|--------------|------|-----------|
| hungerDelta | 배고픔 회복 (음수 = 독) | 생선 20, 별 50, 초콜릿 −10 |
| hpDelta | HP 회복 | 하트 1, 별 3 |
| coinDelta | 코인 획득 | 코인 1 |
| isKey | 열쇠 획득 여부 | true/false |

#### BoxController (상호작용 박스)

- 범위 안에서 **F키** 입력 시 열기 시도
- 열쇠 1개 소모 → 없으면 Locked 애니메이션
- 열리면 Box_Open 애니메이션 + 아이템 `itemCount`개 물리 팝

## 4. 플레이 진행 방법 (현재 버전)

### 씬 실행 조건

1. Unity 에디터에서 `Assets/Scenes/SampleScene.unity` 열기
2. 씬에 다음 컴포넌트가 포함된 오브젝트 필요:
   - **Player**: `PlayerController`, `PlayerAnimationController`, `PlayerAnimationTest`, `PlayerInput`, `Rigidbody2D`, `Collider2D`
   - **GameManager**: `GameManager` (DontDestroyOnLoad)
   - **HungerSystem**: `HungerSystem`
   - **CoinKeySystem**: `CoinKeySystem`
   - **Camera**: `CameraFollow` (target = Player Transform)
   - **HUD Canvas**: `HUDController` + Slider/Text UI 오브젝트 연결

### 플레이 흐름

```
씬 시작
→ GameManager.Start() → Playing 상태 진입 (프롤로그 없음)
→ 플레이어 이동·점프·대시·숙이기 가능
→ 적과 접촉 또는 총알 피격 시 HP 감소
→ 아이템 획득 시 배고픔/HP/코인/열쇠 회복
→ F키로 박스 열기 (열쇠 1개 소모)
→ HP 0 → 사망 애니메이션 (이후 게임오버 처리 없음 — 임시)
→ 배고픔 감소는 진행되나 0이 돼도 추가 패널티 없음 (임시)
```

### 현재 버전 플레이 가능 범위

- 좌우 이동, 점프, 대시, 숙이기, 사다리, 벽 슬라이드
- 사람/고양이 적 회피 및 공격
- 아이템(음식/코인/열쇠) 획득 및 HUD 반영
- 박스 열기 (열쇠 소모)

### 현재 버전 동작하지 않는 것

- 경험치 게이지 (ExpManager 없음)
- 점수 (ScoreManager 없음)
- 저장/불러오기 (SaveManager 없음)
- 스테이지 클리어 / 게임오버 화면
- 배고픔 0 이후 패널티
- 모바일 터치 입력

## 5. 레이어 및 태그 설정 (씬 조립 시 필수)

### 레이어

| 번호 | 이름 | 사용처 |
|------|------|--------|
| 6 | Player | 플레이어 오브젝트 |
| 7 | Ground | 바닥·플랫폼 타일맵 |
| 8 | Enemy | 모든 적 오브젝트 |
| 9 | Item | 아이템 오브젝트 |
| 10 | Interactable | 박스 등 상호작용 오브젝트 |

> Enemy 레이어 오브젝트가 Wall 감지에서 제외되도록 `PlayerController`에 태그 체크 포함.  
> Item 레이어는 적과 충돌하지 않도록 Physics 2D Matrix에서 Enemy↔Item 충돌 비활성화 필요.

### 태그

| 태그 | 사용처 |
|------|--------|
| Player | `FoodItem`, `BoxController`, `HumanEnemy`, `CatEnemy`에서 플레이어 탐색 |
| Enemy | PlayerController 벽 감지 시 제외 대상 식별 |
| Ground | (레이어로 처리, 태그 직접 사용 없음) |

### 사다리 설정

`PlayerController`의 `ladderMask` 필드에 Ladder 레이어를 Inspector에서 지정해야 사다리가 동작.
